# 快速入门

本指南将帮助您快速上手 BioWorkflow，从创建第一个工作流到运行分析。

## 前提条件

- 已完成 [安装指南](installation.md)
- BioWorkflow 服务正在运行
- 现代浏览器（Chrome、Firefox、Edge）

## 1. 访问 BioWorkflow

打开浏览器，访问：

```
http://localhost:8000
```

首次访问时，您将看到登录页面。

## 2. 创建账户

### 注册新账户

1. 点击 "注册" 按钮
2. 填写注册信息：
   - 用户名：唯一标识符（如 `zhangsan`）
   - 邮箱：有效的电子邮件地址
   - 密码：至少8位，包含字母和数字
   - 全名（可选）：显示名称
3. 点击 "注册"

### 首次登录

1. 使用刚注册的用户名和密码登录
2. 系统将自动跳转到 Dashboard

## 3. 探索 Dashboard

Dashboard 是 BioWorkflow 的控制中心，包含：

### 统计卡片

- **活跃流水线**: 当前可用的工作流数量
- **Conda 环境**: 已配置的 Conda 环境数
- **知识库文档**: 知识库中的文档数量
- **MCP 服务**: 已注册的 MCP 服务数

### 最近执行

显示最近运行的工作流及其状态：
- 绿色：成功完成
- 红色：执行失败
- 黄色：正在运行
- 灰色：等待执行

### 快速操作

常用功能的快捷入口：
- 创建新流水线
- 管理 Conda 环境
- 查询知识库
- 配置 MCP 服务

## 4. 创建第一个工作流

### 步骤 1: 进入流水线页面

1. 点击左侧导航栏的 "流水线"
2. 或点击 Dashboard 中的 "创建新流水线" 按钮

### 步骤 2: 使用可视化编辑器

1. 点击 "可视化编辑器" 选项卡
2. 您将看到一个空白的画布

### 步骤 3: 添加节点

1. **添加输入节点**:
   - 点击工具栏的 "输入节点" 按钮
   - 在画布上点击放置
   - 双击节点配置:
     - 标签: "原始数据"
     - 输入文件: `data/*.fastq.gz`

2. **添加处理节点**:
   - 点击 "处理节点" 按钮
   - 放置到画布上
   - 从输入节点拖动连线到处理节点
   - 配置:
     - 标签: "质量控制"
     - 脚本: `fastqc {input} -o {output}`
     - Conda 环境: `qc-env`

3. **添加输出节点**:
   - 点击 "输出节点" 按钮
   - 放置并连接
   - 配置:
     - 标签: "分析结果"
     - 输出目录: `results/`

### 步骤 4: 保存工作流

1. 点击 "保存" 按钮
2. 输入工作流名称: "RNA-seq QC Pipeline"
3. 输入描述: "RNA-seq 数据质量控制流程"
4. 点击 "确认保存"

### 步骤 5: 验证工作流

1. 点击 "验证" 按钮
2. 系统将检查:
   - 所有节点是否正确连接
   - 脚本语法是否正确
   - Conda 环境是否存在
3. 修复任何错误

## 5. 运行工作流

### 方式 1: 从编辑器运行

1. 点击 "运行" 按钮
2. 确认执行参数
3. 系统将跳转到执行状态页面

### 方式 2: 从列表运行

1. 进入 "流水线" 页面
2. 找到您创建的工作流
3. 点击右侧的 "运行" 图标

### 监控执行

执行状态页面显示：

- **实时日志**: 查看每个步骤的输出
- **进度条**: 整体完成百分比
- **资源使用**: CPU、内存、磁盘使用情况
- **预计剩余时间**: 基于当前进度估算

## 6. 查看结果

执行完成后：

1. 点击 "查看结果" 按钮
2. 系统将显示：
   - 输出文件列表
   - 质量控制报告
   - 可视化图表
   - 执行日志摘要

## 常见问题

**Q: 工作流执行失败怎么办？**

A: 1. 查看执行日志，找到错误信息
   2. 检查输入文件是否存在
   3. 验证 Conda 环境配置
   4. 检查脚本语法

**Q: 如何暂停和恢复工作流？**

A: 在执行状态页面，点击 "暂停" 按钮。要恢复，点击 "继续" 按钮。

**Q: 可以同时运行多个工作流吗？**

A: 可以，系统支持并发执行。可以在 Dashboard 的 "最近执行" 区域查看所有运行中的工作流。
